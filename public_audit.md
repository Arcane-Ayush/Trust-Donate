# How to Crack It: The TrustDonate Auditing Process

The entire value of TrustDonate relies on the fact that **the public can mathematically verify NGO expenses without having to trust a central database.** 

If you are asked, *"How does a regular user actually verify a receipt?"*, this document explains the step-by-step process of both the real-world (external) verification and our built-in (internal) Zero-Knowledge Verifier tool.

---

## 1. The External Process (The "Hard" Way)

This is the standard Web3 way of verifying data, proving that our system relies purely on math, not on our own frontend:

1. **The Public Ledger:** A donor goes to the Transparency page and sees an NGO Expense for $500. Attached to it is an immutable Invoice Hash (e.g., `0xabc123...`).
2. **Demanding the Proof:** The donor goes to the NGO's public website or emails them, demanding the PDF receipt for that specific $500 expense.
3. **The Verification:** The NGO hands over the `receipt.pdf`. The donor then takes that PDF and runs it through *any* standard 3rd-party SHA-256 generator. They can use a free online tool, or simply type `shasum -a 256 receipt.pdf` into their Mac/Windows terminal.
4. **The Match:** If the hash generated by the donor exactly matches the `0xabc123...` hash on the Base Sepolia blockchain, the donor has absolute mathematical proof that the NGO didn't photoshop or alter the document *after* committing it to the blockchain.
5. **The Flag:** If the NGO refuses to provide the PDF, OR if the donor hashes the PDF and gets a completely different hash (proving the NGO altered the file before handing it over), the donor returns to the Transparency page and clicks the **Flag Suspicious** button to permanently mark the transaction as fraudulent.

---

## 2. The Internal Process (The "Zero-Knowledge Verifier")

To make this incredibly easy for everyday users, we built an **In-Browser Zero-Knowledge Verifier** directly into the TrustDonate Transparency Dashboard.

During your demo, you can showcase this feature:

1. **The Setup:** Tell the judges, *"We don't expect everyday donors to use a terminal to calculate SHA-256 hashes. So we built a Zero-Knowledge Verifier directly into the app."*
2. **The Action:** Drag and drop an invoice PDF into the Verifier box at the top of the Transparency page.
3. **The Magic:** The browser instantly calculates the cryptographic hash locally—**the PDF never leaves the user's computer, ensuring absolute privacy.**
4. **The Result:** The Verifier instantly scans the live blockchain feed. If it finds a matching hash, it glows **Green** and highlights the exact on-chain expense record in the feed below it. 
5. **The Fraud Detection:** If a user drops a photoshopped PDF into the box, it glows **Red** (Fraud Detected), giving the user the signal to scroll down and click **Flag Suspicious**.

By combining the immutable ledger with an in-browser verifier, TrustDonate completely solves the transparency gap in charitable giving.
